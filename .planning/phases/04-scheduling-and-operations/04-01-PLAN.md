- Define `Event` struct: `SecretName string`, `Status string` (success/failed), `Details string`, `Timestamp time.Time`

- Define `Dispatcher` struct holding `[]Notifier`

- Define `NewNotifiersFromConfig(configs []config.NotifyConfig) []Notifier` factory that maps type strings to implementations: "discord" -> DiscordNotifier, "slack" -> SlackNotifier, default -> GenericNotifier

Create `internal/notify/discord.go`:
- `DiscordNotifier` struct with `webhookURL string`
- `Send` POSTs JSON matching Discord webhook format: `{"embeds": [{"title": "Secret Rotation: {status}", "description": "{secret_name}: {details}", "color": 3066993|15158332, "timestamp": "..."}]}`
- Use `net/http` directly (no external deps per research recommendation)
- 10-second timeout on HTTP client
- Return error if response status >= 400

Create `internal/notify/slack.go`:
- `SlackNotifier` struct with `webhookURL string`
- `Send` POSTs JSON matching Slack webhook format: `{"text": "Secret Rotation {status}: {secret_name}", "blocks": [{"type": "section", "text": {"type": "mrkdwn", "text": "..."}}]}`
- `GenericNotifier` struct with `webhookURL string`
- `Send` POSTs JSON: `{"secret_name": "...", "status": "...", "details": "...", "timestamp": "..."}`
- Both use net/http with 10-second timeout

Tests use httptest.NewServer to capture requests and verify payload format.
