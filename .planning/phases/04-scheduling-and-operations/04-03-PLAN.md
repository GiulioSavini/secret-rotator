  <name>Task 1: Scheduler package with cron runner and file lock</name>
  <files>internal/scheduler/scheduler.go, internal/scheduler/scheduler_test.go</files>
  <behavior>
    - Test 1: Scheduler.AddJob registers a cron job for a secret config with valid cron expression
    - Test 2: Scheduler.AddJob returns error for invalid cron expression
    - Test 3: When cron fires, the rotation function is called with the correct SecretConfig
    - Test 4: Concurrent rotation of the same secret is blocked (second call returns immediately without error)
    - Test 5: Notification is sent after successful rotation with status "success"
    - Test 6: Notification is sent after failed rotation with status "failed" and error details
    - Test 7: Scheduler.Stop cleanly shuts down all cron jobs
  </behavior>
  <action>
Create `internal/scheduler/scheduler.go`:

1. Define `Scheduler` struct:
   - `cron *cron.Cron` (robfig/cron/v3)
   - `rotateFunc func(ctx context.Context, secretCfg config.SecretConfig) error` (injectable for testing)
   - `dispatcher *notify.Dispatcher`
   - `locks sync.Map` (map of secretName -> *sync.Mutex for per-secret locking)
   - `logger *log.Logger`

2. `NewScheduler(rotateFn func(ctx context.Context, cfg config.SecretConfig) error, dispatcher *notify.Dispatcher) *Scheduler`
   - Creates robfig/cron with `cron.WithSeconds()` option disabled (standard 5-field cron), and `cron.WithLogger` for debug output

3. `AddJob(secretCfg config.SecretConfig, cronExpr string) error`
   - Validates cronExpr with `cron.ParseStandard(cronExpr)`
   - Creates a closure that:
     a. Acquires per-secret lock via `sync.Mutex.TryLock()` -- if already locked, log "skipping, rotation in progress" and return
     b. Calls `s.rotateFunc(ctx, secretCfg)`
     c. Sends notification via `s.dispatcher.Send()` with Event{SecretName, status, details, time.Now()}
     d. Releases lock via `defer Unlock()`
   - Adds closure to cron with `s.cron.AddFunc(cronExpr, closure)`

4. `LoadFromConfig(secrets []config.SecretConfig) error`
   - Iterates secrets, calls AddJob for each with non-empty Schedule field

5. `LoadFromLabels(labels []docker.ScheduleLabel, secrets []config.SecretConfig) error`
   - For each label, find matching SecretConfig by name (or apply globally if SecretName is empty)
   - Call AddJob with the label's CronExpr (label schedule overrides config schedule)

6. `Start()` -- calls `s.cron.Start()`
7. `Stop()` -- calls `s.cron.Stop()` and waits for running jobs

Tests: Use a mock rotateFunc (channel-based to detect calls). Use short cron intervals or call job functions directly. Test file lock with goroutines attempting concurrent rotation.
  </action>

  </verify>
  <done>Scheduler registers cron jobs, prevents concurrent rotation of the same secret, sends notifications on rotation outcomes. All tests pass including race detector.</done>
