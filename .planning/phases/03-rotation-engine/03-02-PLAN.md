  <name>Task 1: State types, Executor, and LIFO rollback</name>
  <files>internal/engine/state.go, internal/engine/executor.go, internal/engine/rollback.go, internal/engine/executor_test.go, internal/engine/rollback_test.go</files>
  <behavior>
    - Full pipeline success: init -> backup old secret -> generate new -> apply to DB -> verify DB -> update .env -> restart containers -> health check -> record history -> done
    - Generic provider (Name()=="generic") skips StepApplyDB and StepVerifyDB (no database)
    - Failure at StepApplyDB: rollback calls provider.Rollback, returns error with details
    - Failure at StepVerifyDB: rollback calls provider.Rollback (undo DB), returns error
    - Failure at StepUpdateEnv: rollback calls provider.Rollback (undo DB), returns error
    - Failure at StepRestart: rollback restores old .env content (write old content back), calls provider.Rollback (undo DB), returns error
    - Failure at StepHealthCheck: rollback restarts containers again (with restored .env), restores old .env, calls provider.Rollback (undo DB), returns error
    - History entry recorded on both success (status:"success") and failure (status:"failed", details: error message)
    - Engine verifies env key exists in .env file before starting rotation (ef.Get returns false -> error)
    - Engine backs up full .env file content before any mutation for restore on rollback
    - Rollback failure itself returns a wrapped error with state details for manual recovery
    - DryRun mode: logs what would happen at each step without executing mutations
  </behavior>
  <action>
    **state.go:** Define RotationStep iota enum (StepInit through StepDone as in research). Define RotationState struct tracking: SecretName, CurrentStep, OldSecret, NewSecret, OldEnvContent ([]byte), EnvFilePath, Containers ([]string), Error.

    **executor.go:** Define Engine struct holding dependencies via interfaces:
    - `provider provider.Provider` (the resolved provider for this secret type)
    - `docker docker.Manager` (for container restarts)
    - `history *history.Store` (for recording outcomes, can be nil for dry-run or no-history mode)
    - `timeout time.Duration` (restart timeout)
    - `dryRun bool`

    Constructor: `NewEngine(prov provider.Provider, dockerMgr docker.Manager, histStore *history.Store, timeout time.Duration, dryRun bool) *Engine`

    Main method: `Execute(ctx context.Context, secretCfg config.SecretConfig) error`

    Execute flow:
    1. StepInit: resolve env file path (prefer EnvFile, fall back to first of EnvFiles). Read .env file via envfile.Read(). Verify key exists via ef.Get(secretCfg.EnvKey). If not found, return error "secret key %s not found in %s".
    2. StepBackup: read old secret value, store in state.OldSecret. Read raw file bytes for state.OldEnvContent (os.ReadFile on the env file path).
    3. StepGenerate: build ProviderConfig from secretCfg.Provider map. Call provider.Rotate(ctx, provCfg, oldSecret). Store result.NewSecret in state.
    4. StepApplyDB: skip if provider.Name() == "generic" (generic has no DB). Otherwise this was handled inside provider.Rotate already -- this step is implicit. Move to StepVerifyDB.
    5. StepVerifyDB: skip if provider.Name() == "generic". Call provider.Verify(ctx, provCfg, newSecret).
    6. StepUpdateEnv: call ef.Set(envKey, newSecret) + ef.WriteAtomic(). If multiple env files (EnvFiles), iterate and update all.
    7. StepRestart: call docker.RestartInOrder(ctx, mgr, secretCfg.Containers, timeout). Store container names in state.Containers.
    8. StepHealthCheck: already handled by RestartInOrder (it waits for healthy after each restart). This step is implicit.
    9. StepRecord: if history store is not nil, append HistoryEntry with status "success".
    10. StepDone: return nil.

    On any error at step >= StepGenerate (where provider.Rotate was called, meaning DB may have been mutated): call rollback(ctx, state, provider, dockerMgr, timeout). Then record failure in history.

    IMPORTANT: The provider.Rotate method handles both generating the new password AND applying it to the DB in one call (per the Provider interface design). StepApplyDB and StepVerifyDB are conceptual steps -- Rotate does the DB work, Verify confirms it. For the generic provider, Rotate just generates (no DB), and Verify is a no-op.

    For DryRun: log each step with "[DRY RUN] Would {action}" and skip actual mutations. Return nil.

    **rollback.go:** `func rollback(ctx context.Context, state *RotationState, prov provider.Provider, dockerMgr docker.Manager, envFilePath string, timeout time.Duration) error`

    LIFO rollback based on state.CurrentStep:
    - If step >= StepRestart and containers were restarted: restore old .env content first (os.WriteFile), then RestartInOrder again to pick up old values.
    - If step >= StepUpdateEnv: restore old .env content by writing state.OldEnvContent back to the file.
    - If step >= StepGenerate and provider.Name() != "generic": call provider.Rollback(ctx, cfg, state.OldSecret) to restore old DB password.

    Collect all rollback errors. If any rollback step fails, wrap with context: "rollback failed at step X: {err}; current state: DB has {new|old} password, .env has {new|old} password, containers {restarted|not restarted}".

    Write tests FIRST. Use mock provider (implement Provider interface inline in test file with configurable failures), mock docker manager (record calls, optionally return errors), and real envfile operations on temp files. Test scenarios:
    - Happy path: all steps succeed, history records "success"
    - Provider.Rotate fails: no rollback needed (DB not mutated), history records "failed"
    - Provider.Verify fails: rollback calls provider.Rollback
    - envfile.WriteAtomic fails: rollback calls provider.Rollback
    - RestartInOrder fails: rollback restores .env + calls provider.Rollback
    - Generic provider skips DB steps
    - DryRun mode does not mutate anything
    - Key not found in .env file returns error before any mutation
  </action>
