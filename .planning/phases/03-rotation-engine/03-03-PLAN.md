    - Rotate connects as admin (cfg.Username + cfg.Options["password"]), executes ALTER USER on cfg.Options["target_user"] (or cfg.Username if not set), returns Result with old+new
    - Verify connects as target user with new password, runs db.Ping()
    - Rollback connects as admin, executes ALTER USER with old password
    - Connection uses DSN format: "user:pass@tcp(host:port)/"
    - Uses cfg.Port default 3306 if 0

    PostgreSQL:
    - PostgresProvider.Name() returns "postgres"
    - Rotate connects as admin via pgx, executes ALTER ROLE with pgx.Identifier sanitization for role name
    - Verify connects as target user with new password, runs conn.Ping()
    - Rollback connects as admin, executes ALTER ROLE with old password
    - Connection uses connstring: "postgres://user:pass@host:port/database"
    - Uses cfg.Port default 5432 if 0

    Redis:
    - RedisProvider.Name() returns "redis"
    - Rotate connects with current password, executes CONFIG SET requirepass + CONFIG REWRITE
    - If CONFIG REWRITE fails, immediately rolls back CONFIG SET to old password and returns error
    - Verify creates new client with new password, runs client.Ping()
    - Rollback connects with old password (or new if old fails), executes CONFIG SET requirepass to old + CONFIG REWRITE
    - Uses cfg.Port default 6379 if 0
  </behavior>
  <action>
    First, add dependencies:

    **mysql.go:** MySQLProvider struct (empty).
    - `Rotate`: generate password via GeneratePassword. Build DSN from cfg (admin credentials). Open sql.DB. Execute `ALTER USER '%s'@'%%' IDENTIFIED BY '%s'` with target user and new password. Passwords are base64 URL-safe (only [A-Za-z0-9_-]) so no SQL injection risk. Close DB. Return Result.
    - `Verify`: build DSN with target user + secret. Open sql.DB. db.Ping(). Close.
    - `Rollback`: same as Rotate but with oldSecret instead of generated password.
    - Helper: `mysqlDSN(user, pass, host string, port int) string` builds DSN. Default port 3306.
    - Target user: cfg.Options["target_user"] if set, otherwise cfg.Username.
    - Admin password: cfg.Options["password"] or cfg.Options["password_env"] (read from os.Getenv).

    **postgres.go:** PostgresProvider struct (empty).
    - `Rotate`: generate password. Build connstring. pgx.Connect(ctx, connStr). Execute `ALTER ROLE %s WITH PASSWORD '%s'` using pgx.Identifier{targetUser}.Sanitize() for role name. conn.Close(ctx). Return Result.
    - `Verify`: connect as target user with new password. conn.Ping(ctx). Close.
    - `Rollback`: same as Rotate but with oldSecret.
    - Helper: `pgConnStr(user, pass, host string, port int, database string) string`. Default port 5432.
    - Target user: cfg.Options["target_user"] if set, otherwise cfg.Username.
    - Admin password: cfg.Options["password"] or cfg.Options["password_env"] (read from os.Getenv).

    **redis.go:** RedisProvider struct (empty).
    - `Rotate`: generate password. Create redis.NewClient with current password. ConfigSet("requirepass", newPass). ConfigRewrite(). If ConfigRewrite fails, immediately ConfigSet("requirepass", currentSecret) to rollback, then return wrapped error. Close client. Return Result.
    - `Verify`: create new client with new password. Ping(ctx). Close.
    - `Rollback`: try connecting with old password first. If that fails, try new password. ConfigSet("requirepass", oldSecret). ConfigRewrite(). Close.
    - Helper: `redisAddr(host string, port int) string`. Default port 6379.

    **Tests:** Write unit tests using interfaces/mocks. For each provider, test the method signatures and error handling. Since actual DB connections cannot be unit tested, the tests should:
    - Verify Name() returns correct string
    - Verify the provider implements the Provider interface (compile-time check via `var _ Provider = (*MySQLProvider)(nil)`)
    - For actual connection tests, use build tag `//go:build integration` and document that these need real services via testcontainers

    For unit-testable logic: test DSN/connection string building helpers, test that Rotate calls GeneratePassword, test error wrapping.

    NOTE: Do NOT use parameterized queries for ALTER USER/ROLE password -- MySQL and PostgreSQL do not support `?` placeholders in DDL IDENTIFIED BY / WITH PASSWORD clauses. The generated passwords are base64 URL-safe ([A-Za-z0-9_-] only) which eliminates SQL injection risk by design.
