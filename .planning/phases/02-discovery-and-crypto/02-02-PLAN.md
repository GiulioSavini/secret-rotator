    First, add the direct dependency: `go get golang.org/x/crypto@latest` (already transitive via compose-go, but promote to direct).

    **internal/crypto/crypto.go**: Implement exactly as specified in research Pattern 3:
    - Constants: argonTime=1, argonMemory=64*1024, argonThreads=4, argonKeyLen=32, saltLen=16
    - `DeriveKey(passphrase, salt []byte) []byte` -- calls `argon2.IDKey`
    - `Encrypt(plaintext, passphrase []byte) ([]byte, error)` -- generates random salt (16 bytes) and nonce (12 bytes) via crypto/rand, derives key, creates AES-256-GCM cipher, returns salt+nonce+ciphertext
    - `Decrypt(data, passphrase []byte) ([]byte, error)` -- extracts salt, derives key, extracts nonce, opens GCM ciphertext. Returns `ErrDecryptionFailed` on wrong passphrase, `ErrDataTooShort` if data too small.
    - Define sentinel errors: `var ErrDecryptionFailed = errors.New("decryption failed: wrong passphrase or corrupted data")`, `var ErrDataTooShort = errors.New("encrypted data too short")`

    **internal/history/types.go**: Define types per research Pattern 4:
    - `HistoryEntry` with SecretName, RotatedAt (time.Time), OldValue, NewHash, Status, Details
    - `HistoryFile` with Version (int) and Entries ([]EncryptedEntry)
    - `EncryptedEntry` with Data (string, base64) and CreatedAt (string, ISO 8601)

    **internal/history/store.go**: Implement `Store` struct:
    - Fields: `path string` (file path), `passphrase []byte`, `derivedKey []byte` (cached), `salt []byte` (cached)
    - `NewStore(path string, passphrase []byte) *Store`
    - `Append(entry HistoryEntry) error`: marshal entry to JSON, encrypt with passphrase, base64-encode, read existing file (or create new with Version=1), append EncryptedEntry, write file atomically (use same temp+rename pattern from envfile).
    - `List() ([]HistoryEntry, error)`: read file, for each encrypted entry base64-decode + decrypt. **Key caching anti-pattern avoidance:** For List, we cannot cache a single derived key because each entry has its own random salt (from Encrypt). Instead, to enable key caching, add a `EncryptWithKey(plaintext, key, salt []byte)` and `DecryptWithKey(data, key []byte)` pair that separates key derivation from encryption. The Store derives the key once from a fixed salt stored in the history file header, then uses EncryptWithKey/DecryptWithKey for all entries. Update HistoryFile to include a `Salt` field (base64). On first Append to a new file, generate a random salt and store it. Subsequent appends reuse the same salt. This way DeriveKey is called once per command invocation.
    - Handle missing file gracefully: return empty slice, nil error.

    Write tests RED first, then implement GREEN. Use testify and t.TempDir().
