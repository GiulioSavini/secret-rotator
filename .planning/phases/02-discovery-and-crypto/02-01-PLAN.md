    - TestAuditStrength_Good: "MyP@ssw0rd!2024x" (16 chars, 4 classes) -> StrengthGood
    - TestAuditStrength_Strong: 32+ char random string with mixed classes -> StrengthStrong
    - TestAuditStrength_DefaultDetection: all entries in CommonDefaults map -> StrengthWeak with "common default" in Issues
    - TestEntropy: entropy calculation returns correct bits for known charset/length combinations
  </behavior>
  <action>
    Create `internal/discovery/` package with three files:

    **patterns.go**: Define `SecretPattern` struct with `Suffix` and `Type` fields. Define `DefaultPatterns` slice covering: `_PASSWORD`, `_PASSWD`, `_SECRET`, `_KEY`, `_TOKEN`, `_API_KEY`, `_APIKEY`, `_AUTH`, `_CREDENTIAL`, `_PRIVATE_KEY`. Define `ExactPatterns` slice: `SECRET_KEY`, `JWT_SECRET`, `SESSION_SECRET`, `DATABASE_URL`, `REDIS_URL`, `MONGO_URI`. Define `CommonDefaults` map with ~30 known weak passwords per research (changeme, password, secret, admin, root, default, test, example, 12345678, etc.).

    **scanner.go**: Define `Scanner` struct holding suffix and exact patterns (use defaults if none provided). Define `DiscoveredSecret` struct with fields: `Key string`, `Type string`, `Source string` (file path), `Value string` (for internal strength check only -- NEVER expose in output), `Strength StrengthResult`, `FileReferenced bool`. Implement `NewScanner() *Scanner` using default patterns. Implement `classifyKey(key string) (secretType string, fileReferenced bool)` -- uppercase the key, check exact matches first, then suffix matches. If key ends with `_FILE` AND a pattern matches the part before `_FILE`, return the type with fileReferenced=true. Implement `ScanFile(ef *envfile.EnvFile) []DiscoveredSecret` -- iterate ef.Keys(), classify each, if classified call AuditStrength on the value (skip strength for fileReferenced), build DiscoveredSecret. Implement `ScanFiles(files []*envfile.EnvFile) []DiscoveredSecret` for convenience.

    **strength.go**: Define `Strength` type (int) with constants `StrengthWeak` (0), `StrengthFair` (1), `StrengthGood` (2), `StrengthStrong` (3). Add `String()` method returning "weak"/"fair"/"good"/"strong". Define `StrengthResult` struct with `Score Strength`, `Entropy float64`, `Length int`, `Issues []string`. Implement `AuditStrength(value string) StrengthResult`:
    1. Check empty -> weak, issue "empty password"
    2. Check CommonDefaults map (case-insensitive) -> weak, issue "common default password"
    3. Calculate entropy: count character classes (lower, upper, digit, symbol), determine charset size, entropy = length * log2(charset_size)
    4. Check length: <8 weak, 8-15 fair, 16-31 good, 32+ strong
    5. Check character classes: <2 classes adds issue "limited character variety"
    6. Final score = min(length_score, entropy_score) with adjustments for class count
    Scoring: entropy <28 bits = weak, <36 = fair, <60 = good, 60+ = strong.

    Write tests RED first, then implement GREEN. Use testify assert/require per project convention.
