<task type="auto">
  <name>Task 2: Dockerfile and .dockerignore</name>
  <files>Dockerfile, .dockerignore</files>
  <action>
Create a multi-stage Dockerfile:

**Stage 1 (builder):** Use `golang:1.25-alpine` as build image. Set `CGO_ENABLED=0`. Copy `go.mod`, `go.sum` first and run `go mod download` for layer caching. Then copy the rest of the source. Build with:
```
go build -ldflags "-s -w -X github.com/giulio/secret-rotator/internal/cli.version=${VERSION:-dev} -X github.com/giulio/secret-rotator/internal/cli.commit=${COMMIT:-none} -X github.com/giulio/secret-rotator/internal/cli.date=${DATE:-unknown}" -o /rotator ./cmd/rotator
```
Use ARG for VERSION, COMMIT, DATE so they can be passed at build time via `--build-arg`.

**Stage 2 (runtime):** Use `gcr.io/distroless/static-debian12:nonroot` as the runtime image. This provides:
- No shell (security hardened)
- Runs as nonroot user (UID 65534) by default
- Minimal attack surface
- ca-certificates included for HTTPS webhook calls

Copy the binary from builder. Set `ENTRYPOINT ["/rotator"]`.

Document expected volume mounts in LABEL instructions:
- `/var/run/docker.sock` for Docker socket access
- `/config` for rotator.yml and .env files

Add LABELs for OCI image metadata: `org.opencontainers.image.source`, `org.opencontainers.image.title`, `org.opencontainers.image.description`.

Create `.dockerignore` excluding: `.git`, `.planning`, `*.md` (except rotator.example.yml is not .md so it passes), `.env*`, `rotator.yml`, and the `rotator` binary directory if it exists.
  </action>

  </verify>
  <done>Docker image builds successfully. `docker run --rm secret-rotator:test version` prints "rotator version test". Container runs as nonroot user (UID 65534). Image size is under 30MB.</done>
