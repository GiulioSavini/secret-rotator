<task type="auto" tdd="true">
  <name>Task 2: Compose file parsing for dependency ordering</name>
  <files>internal/docker/compose.go, internal/docker/compose_test.go</files>
  <behavior>
    - TestLoadDependencyOrderSimple: Compose file with A depends_on B returns [B, A]
    - TestLoadDependencyOrderChain: A->B->C returns [C, B, A]
    - TestLoadDependencyOrderDiamond: A->B, A->C, B->D, C->D returns D first, A last
    - TestLoadDependencyOrderNoDeps: Services with no depends_on return in any order
    - TestLoadDependencyOrderCycle: Circular depends_on returns an error (not infinite loop)
    - TestLoadDependencyOrderFilterServices: Given a list of target service names, return only those services in dependency order (not all services in the compose file)
  </behavior>
  <action>
    1. Install compose-go dependency: `go get github.com/compose-spec/compose-go/v2@latest`
    2. Create `internal/docker/compose.go`:
       - LoadDependencyOrder(composePath string) ([]string, error):
         - Use compose-spec/compose-go to parse the compose file
         - NOTE: The compose-go API may differ from research examples. Check the actual API:
           - Try: loader.Load with types.ConfigDetails{ConfigFiles: []types.ConfigFile{{Filename: composePath}}}
           - Or: cli.NewProjectOptions approach from research
           - The key is to get a *types.Project with Services that have DependsOn maps
         - Extract depends_on relationships into adjacency list
         - Run topological sort (Kahn's algorithm) to produce dependency-first ordering
         - Return error if cycle detected
       - FilterDependencyOrder(allOrder []string, targets []string) []string:
         - Given the full dependency order and a subset of target service names
         - Return only the targets, preserving their relative dependency order
       - topoSort(deps map[string][]string, allNodes []string) ([]string, error):
         - Kahn's algorithm implementation
         - Track in-degree for each node
         - BFS from zero in-degree nodes
         - Return error if not all nodes processed (cycle detected)
    3. Create `internal/docker/compose_test.go`:
       - Create test compose YAML files in t.TempDir()
       - Test all behaviors above
       - Use minimal compose files (just service names and depends_on, no images needed for parsing)
       - Note: compose-go may require `image` or `build` fields to be valid. If so, add dummy `image: scratch` to test services.
  </action>
